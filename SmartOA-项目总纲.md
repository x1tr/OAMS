 # SmartOA — 企业智慧办公管理系统 · 项目总纲

> **实训周期**：10天 | **技术栈**：Vue.js + SpringBoot + SpringSecurity + Nacos + MyBatis + Redis + Elasticsearch + SpringAI + RedisStack | **团队**：武汉理工大学实训班

---

## 一、项目概要

一个带 **AI智能问答** 的企业OA系统。员工能查公告、搜文档、提交审批、问AI；管理员能管用户、审流程。**核心卖点**：AI不是外挂——你问"请假要什么材料"，它从知识库里检索制度文档后回答你，而不是返回一堆链接。

### 用户角色（3种）

| 角色 | 能干什么 |
|------|----------|
| **普通员工** | 看公告、搜文档、提交请假申请、AI问答 |
| **部门经理** | 上面的 + 审批下级申请 |
| **系统管理员** | 管用户/管部门/管公告/看所有数据 |

### 技术全景

```
Vue.js 3 (Element Plus)
        │
        ▼
Gateway (Spring Cloud Gateway)  ← JWT鉴权 + 路由转发
        │
   ┌────┼────┐
   ▼    ▼    ▼
emp-svc  admin-svc   ai-svc
员工    管理+审批    AI客服
MyBatis MyBatis   Spring AI 3.x
  ES      Redis     Ollama
        │  RedisStack
        ▼
MySQL · Redis · ES · Nacos · Ollama
```

## 三、模块拆分与分工建议

假设8-10人团队，分4组：

| 组 | 人数 | 负责微服务 | 核心任务 | 难度 |
|------|------|------|------|------|
| **后端A** | 2-3人 | oa-emp-service | 员工CRUD + ES搜索 | ⭐⭐ |
| **后端B** | 2-3人 | oa-admin-service | RBAC权限 + 审批 + 公告 + Redis | ⭐⭐⭐ |
| **后端C** | 2人 | oa-ai-service | Ollama对话 + RAG + 向量库 | ⭐⭐⭐ |
| **前端** | 2-3人 | Vue项目 | 所有页面 + 对接三个后端 | ⭐⭐ |
