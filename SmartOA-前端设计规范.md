---

## 文档信息

| 项目 | 内容 |
|------|------|
| **技术栈** | Vue.js 3 + Element Plus + Pinia + Vue Router + Axios + Vite |

## 一、设计定位

SmartOA 的使用场景是**每天8小时的工作环境**，不是炫技的营销页面。设计要做到：

- **不打扰** — 颜色不刺眼，动画不频繁，让用户专注于工作内容
- **有秩序** — 信息层级一目了然，操作路径最短
- **一个记忆点** — 侧栏导航的"呼吸指示器"，是唯一的花活，用完就收

**刻意避开的三个坑**：
- 不用暖色调奶油底+高衬线字体（那是杂志风，不是OA）
- 不用纯黑底+荧光绿（那是极客风，太压抑）
- 不用密集报纸排版（那是信息焦虑风）

## 二、Design Token

### 2.1 色彩体系

```
PRIMARY（品牌蓝）— 导航、按钮、链接
  #E8F0FE  浅色背景
  #2563EB  主色 ★
  #1D4ED8  深色/hover

ACCENT（审批绿）— 通过、完成、成功
  #ECFDF5  浅色背景
  #10B981  主色 ★
  #059669  深色

AI（智慧紫）— AI功能专属标识
  #F5F3FF  浅色背景
  #7C3AED  主色 ★

NEUTRAL（灰阶）
  #FFFFFF  白底
  #F8FAFC  页面底色
  #F1F5F9  卡片底色
  #E2E8F0  边框
  #94A3B8  次要文字
  #64748B  正文
  #334155  标题
  #0F172A  侧栏深色底

FUNCTIONAL
  #EF4444  危险/删除/驳回
  #F59E0B  警告/待处理
  #3B82F6  链接/信息
```

**用色规则**：
- 60% 灰色系（背景/卡片/边框）、30% 蓝色系（交互元素）、10% 紫色/绿色（AI/审批标识）
- AI 相关的按钮/气泡/图标**必须用紫色**，建立视觉识别
- 审批通过用绿色、驳回用红色，不要混

### 2.2 字体

```
标题：PingFang SC → Microsoft YaHei → system-ui
      24px(页面标题) / 20px(卡片标题) / 16px(区块标题)
      字重 500-600

正文：PingFang SC → Microsoft YaHei → system-ui
      14px / 字重 400 / 行高 1.6

辅助：SF Mono → Consolas → monospace
      12px / 用于时间戳、编号、工号
```

不用 Web Font，系统原生字体在 Win/Mac 上表现足够好，且不影响首屏加载。

### 2.3 间距（4px基准）

```
 4px  图标内边距、紧凑标签
 8px  组件内间距
12px  表单项间距
16px  卡片内边距 ★ 最常用
24px  区块间距、页面内边距
32px  大区块分隔
```

### 2.4 圆角与阴影

```
圆角：4px(输入框/标签)  6px(卡片/按钮)  10px(弹窗)
阴影（仅3级，克制）：
  sm: 0 1px 3px rgba(0,0,0,0.06)    卡片悬浮
  md: 0 4px 12px rgba(0,0,0,0.08)   下拉菜单
  lg: 0 8px 24px rgba(0,0,0,0.12)   弹窗
```

### 6.3 SSE 接收（composable）

```typescript
// composables/useSSE.ts
export function useSSE() {
  const text = ref('')
  const isStreaming = ref(false)
  let controller: AbortController

  async function sendMessage(question: string) {
    text.value = ''
    isStreaming.value = true
    controller = new AbortController()

    const response = await fetch('/api/ai/chat/stream', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${token}` },
      body: JSON.stringify({ question }),
      signal: controller.signal
    })

    const reader = response.body!.getReader()
    const decoder = new TextDecoder()

    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      const chunk = decoder.decode(value)
      // 解析 SSE 格式: "data: 文本内容\n\n"
      const lines = chunk.split('\n')
      for (const line of lines) {
        if (line.startsWith('data: ')) {
          text.value += line.slice(6)
        }
      }
    }
    isStreaming.value = false
  }

  function stop() { controller?.abort() }

  return { text, isStreaming, sendMessage, stop }
}
```

## 七、动效规范（克制）

| 场景 | 动效 | 时长 |
|------|------|------|
