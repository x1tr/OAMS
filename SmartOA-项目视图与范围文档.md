## 一、项目背景

### 1.1 为什么做这个项目

传统OA系统存在三个典型问题：

| 问题 | 现状 | 后果 |
|------|------|------|
| **找信息太难** | 公告、文档散落在各处，想找"去年的考勤规定"得翻半天 | 效率低下 |
| **审批不灵活** | 请假只能从固定模板选，员工想申请"远程办公半天"找不到对应类型 | 流程僵化 |
| **重复问题多** | HR/行政每天回答100遍"年假怎么申请""报销要什么材料" | 人力浪费 |

本项目用 **AI + 全文搜索 + 灵活审批** 三条线解决这些问题。

### 1.2 技术栈全景

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端 | Vue.js 3 + Element Plus + Vite | Vue 3.4+ |
| 网关 | Spring Cloud Gateway + Nacos | Gateway 3.x, Nacos 1.1.3+ |
| 业务服务 | SpringBoot + MyBatis + SpringSecurity | Boot 2.x |
| AI服务 | SpringBoot + Spring AI 1.0 + WebFlux | **Boot 3.4.5**（独立栈） |
| 数据存储 | MySQL 8.0 + Redis + Elasticsearch 7.13 | — |
| AI引擎 | Ollama（轻量模型 + 内存向量库） | qwen2.5:0.5b |
| 构建工具 | Maven 3.9 + JDK 21 | — |

### 1.3 四个微服务（基于已有POM）

## 二、项目范围

### 2.1 要做的（In Scope）

#### 模块一：员工管理（emp-service）

```
功能清单：
├── 员工增删改查（姓名/工号/部门/职位/手机/邮箱/状态）
├── 分页查询 + 多条件筛选
├── ES关键词搜索（IK中文分词，输入"技术"能搜出"技术部"所有人）
├── 部门树形结构维护
└── 数据脱敏（手机号/身份证自动打码）
```

#### 模块二：权限与审批（admin-service）

```
功能清单：
├── JWT登录认证（SpringSecurity）
├── RBAC三级角色：管理员 / 部门经理 / 普通员工
├── 路由级 + 按钮级权限控制
├── Redis缓存（用户信息/角色权限/部门树/Session）
│
├── ★ 自定义请假系统（灵活制，非固定模板）
│   ├── 员工自行填写：
│   │   · 请假类型：自由文本输入（如"远程办公"、"产检"、"考证"）
│   │   · 开始时间 / 结束时间
│   │   · 请假事由：自由文本（支持详细描述）
│   │   · 附件上传（可选，如医院证明照片）
│   ├── 提交 → 部门经理审批 → 通过/驳回
│   └── 驳回时经理必须填写驳回理由
│
├── 公告管理（发布/编辑/删除/ES搜索）
└── 操作日志（谁在什么时间做了什么）
```

**★ 自定义请假的设计理念**：

传统OA的请假模块通常是下拉选择"年假/事假/病假/婚假"等固定类型。但在实际工作中，员工的请假需求千差万别——"我需要下午请2小时去接孩子"、"我要远程办公三天"、"我要请学术会议假"——固定模板兜不住。

**所以我们的做法**：请假类型不设下拉框，改为自由文本输入。员工在输入框里写什么就是什么。这样做的好处：

- 员工不需要在有限选项中纠结"我的情况属于哪一类"
- 经理审批时看到的是员工真实的诉求，而非被格式化后的标签
- 系统不需要维护"假期类型字典表"，减少配置
- 灵活适应任何企业场景——同一个系统，A公司员工写"年假"，B公司员工写"调休"，都能跑

> **实训价值**：这个设计让学生理解——软件应该适配人的行为，而不是让人去适配软件的选项。

#### 模块三：AI智能客服（ai-service）

```
功能清单：
├── 知识库管理（上传公司制度文档 → 切片 → 向量化）
├── RAG问答流程（问→向量检索→组装上下文→AI生成→流式输出）
├── SSE流式对话（前端逐字渲染，像ChatGPT一样）
├── 多轮对话上下文记忆（同一个会话内记住前面的问答）
└── 建议问题推荐（"你可能想问：年假怎么申请？报销流程是什么？"）
```

#### 模块四：前端门户（Vue.js 3）

```
页面清单：
├── 登录页        品牌展示 + 登录表单
├── 工作台首页    统计卡片 + 待办列表 + 快捷入口
├── 员工管理页    搜索栏 + 表格 + 分页 + 编辑弹窗
├── 审批中心页    我的申请 / 待我审批 两个Tab
├── 公告列表页    ES搜索 + 公告详情
├── AI客服对话页  历史会话 + 聊天区 + 流式渲染
└── 系统设置页    用户管理 / 角色管理（管理员可见）
```

### 2.2 不做的（Out of Scope — V1.0）

- ❌ 3级以上复杂审批流（只做员工→经理两级）
- ❌ 数据大屏 / ECharts图表看板
- ❌ 移动端原生APP（仅桌面端Web）
- ❌ SSO单点登录 / OAuth2.0
- ❌ 多语言国际化
- ❌ 分布式事务
- ❌ CI/CD自动部署流水线
- ❌ 高并发压测与性能调优

    ▼

    ▼
